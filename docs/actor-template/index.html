Version # The creation of a user's comment

Summary The actor creates a new comment, usually in reaction to a post or another comment
Priority Must have
Pre Condition none
Main Scenario (MS)
  1. actor clicks on 'reply' button
  2. system opens a new input box somewhere below the reply button
  3. actor provides their reaction text in the input box
  4. actor clicks on the 'post' button
  5. system checks the comment for no-no words
  6. system replaces filter words with their counterparts
  7. system uploads the reply
Post Condition (A0) system now contains a new comment
Alternative Scenario (A1)
Trigger: After MS4: [CONNECTION LOST]
Steps:
  1. system shows a notification that the comment could not be sent due to lack of connection
  2. system fades the notification over a minute of time
  3. Use Case continues after MS3
Alternative Scenario (A2)
Trigger: After MS4: [MESSAGE NOT SENT]
Steps:
  1. system shows a notification that the comment could not be sent due to unknown reasons
  2. system fades the notification over a minute of time
  3. Use Case continues after MS3
Alternative Scenario (A3)
Trigger: After MS5: [NO-NO WORD DETECTED]
Steps:
  1. system shows a notification that {NO-NO WORD} was detected
  2. system shows a notification that if it was a false positive, it's fine to post, but if it's not and the user does it, they'll be punished
  3. actor posts comment
  4. Use Case continues after MS5
